# 项目开发计划表：2D角色模板驱动系统

## 1. 计划目标
- 以“可编程演出 Runtime（PixiJS + Spine Runtime + GSAP + DSL + Timeline）”为第一阶段交付核心
- 用最小 Demo 资产闭环验收 M1，避免“无资产无法验收”
- 以“资产规范化（Prefab 规范）+ 命令语义 + 可观测与调试面板”为工程化主线，保证可扩展与可复用

## 3. 最小 Demo 资产准备（贯穿前置）

### 3.1 资产清单（必须具备）
- 背景图：`bg_classroom_day`（任意一张）
- Spine 角色 prefab：`girl_01_spine`
  - 资源：`skeleton.json`（或 binary）、`skeleton.atlas`、贴图 png
  - 最低动画：`idle`、`talk`（或 mouth track 配置）
- 示例 Scene DSL JSON：覆盖 SetBackground/Spawn/SetEmotion/Talk/Camera.ToShot/Dialogue.Say

### 3.2 Prefab 规范检查点（MUST）
- 动画映射：`idle` 必有；`talk` 或 mouth track 二选一
- 关键绑定：bones（head/chest/root 或 hips）至少 3 个；slots（mouth/eyeL/eyeR）至少 2 个
- MixPolicy：默认值可用，且支持按 prefab 覆盖

## 5. 验收用例（按功能路径）

### 5.1 M1 验收用例（必须全部通过）
- 用示例 DSL 启动场景：背景显示
- `Character.Spawn` 成功：角色出现在 CharactersLayer
- `Character.SetEmotion(happy)` 生效：表情/姿态有变化（允许降级）
- `Character.Talk(true)` 生效：talk 动作或 mouth track 运行（允许降级）
- `Camera.ToShot(close)` 生效：镜头平滑推进
- `Dialogue.Say` 生效：字幕显示在 UI 层

### 5.2 M2 验收用例（关键）
- seek 到任意时间点不崩溃
- seek 后角色动作/情绪/镜头能恢复到“目标最终状态”
- Dialogue/Shake 等一次性命令不因 seek 被重复触发（默认）
- 命令失败可在控制台看到：cmd、args、错误码、是否继续

## 6. 风险清单与对策

| 风险 | 触发条件 | 影响 | 对策 |
| --- | --- | --- | --- |
| Spine 授权不清晰 | 使用示例资源/商用发布前未确认许可 | 交付/发布卡住 | 项目启动阶段完成授权闭环检查（Editor/Runtime/资产） |
| 资源不统一 | 不同角色命名/动画缺失 | 复用失败、返工 | Prefab 规范（MUST）+ 启动校验 + bindings 映射 |
| Talk/Emotion 资源不足 | 只有 idle 无表情 | 体验不足 | 三档降级策略 + 后续补齐资源或情绪动画 |
| Timeline seek 还原困难 | 事件与状态混杂 | 跳过/快进不稳定 | 状态命令/一次性命令分类 + 状态重放 |
